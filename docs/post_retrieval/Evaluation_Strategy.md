# Evaluation Strategy: Generation Stage

This document defines the metrics and methodology used to evaluate the answers generated by the RAG pipeline.

## Metrics

We use a combination of automated metrics to assess different aspects of the generated answers.

### 1. Semantic Answer Similarity (SAS)
-   **Definition**: Measures the semantic similarity between the generated answer and the ground truth.
-   **Calculation**: Uses a SentenceTransformer model (e.g., `all-MiniLM-L6-v2`) to encode both answers into embeddings and calculates the cosine similarity.
-   **Scale**: 0.0 to 1.0 (Higher is better).

### 2. ROUGE-L
-   **Definition**: Measures the longest common subsequence of words between the generated answer and the ground truth.
-   **Scale**: 0.0 to 1.0 (Higher is better).

### 3. LLM-as-a-Judge (Factual Accuracy)
-   **Definition**: A powerful secondary LLM pass that acts as a human proxy to determine if the generated answer is factually correct relative to the ground truth.
-   **Methodology**:
    -   The judge receives the ground truth and the generated answer.
    -   **Few-Shot Instructions**: Uses task-specific examples to improve consistency across different model sizes.
    -   **Binary Classification**: Outputs a 1 (Correct) or 0 (Incorrect).
-   **Auditability**: The pipeline captures the `llm_judge_raw_response`, allowing for manual verification of the judge's reasoning.
-   **Scale**: Binary (0 or 1), aggregated as Accuracy.

## Evaluation Workflow

1.  **Input**: A JSON file containing retrieved results and context for a set of questions.
2.  **Generation**:
    -   The pipeline generates an answer using the current RAG configuration.
    -   (Optional) A baseline answer is generated if configured.
3.  **Scoring**:
    -   Calculate SAS and ROUGE-L against the `text_answer` in the ground truth.
    -   **LLM Judge**: Performs qualitative validation of the factuality of the answer.
4.  **Logging**:
    -   All metrics and raw judge responses are saved to `data/intermediate/post_retrieval/generation_evaluation.json`.
    -   The `llm_judge_raw_response` provides a source of truth for debugging why a question failed.

## Usage

To run the generation evaluation:

```bash
python -m src.post_retrieval.scripts.run_evaluate_generation --representation <representation_type>
```
